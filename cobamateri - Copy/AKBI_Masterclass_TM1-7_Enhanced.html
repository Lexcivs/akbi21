Apa itu Akuntansi Biaya? DEFINISI

Akuntansi biaya adalah cabang akuntansi yang berfokus pada pengumpulan (collecting), pengukuran (measuring), dan pelaporan (reporting/finishing) informasi biaya untuk membantu manajemen dalam perencanaan dan pengendalian operasi perusahaan.

⚡ Sering Keluar di UTS
Tujuan akuntansi biaya: Menyediakan informasi bagi MANAJEMEN (internal) untuk perencanaan dan pengendalian operasi perusahaan. Bukan untuk investor, pajak, atau bank — itu tujuan akuntansi keuangan!

Karakteristik Akuntansi Manajemen yang bukan dimiliki akuntansi keuangan: fokus internal, informasi detail, menekankan masa depan. Yang bukan karakteristik akuntansi manajemen = FOKUS EKSTERNAL.

Laporan akuntansi manajemen: tergantung apakah metode bermanfaat atau tidak bagi perusahaan (bukan harus ikut GAAP).

⚡ Pernyataan yang BENAR tentang perilaku biaya
✅ "Dalam jangka panjang, semua biaya bersifat variabel" — BENAR
❌ "Biaya tetap secara per unit juga memiliki sifat yang tetap" — SALAH (per unit berubah!)
❌ "Biaya variabel per unit akan menurun seiring peningkatan produksi" — SALAH (tetap per unit)
BBL memiliki Perilaku... KEY ANSWER

Bahan baku langsung memiliki perilaku: Variabel per unitnya DAN variabel secara total

Karena setiap unit produk butuh jumlah bahan baku yang sama (tetap per unit), dan total bahan baku berubah sesuai jumlah produksi.

⚠️ Perhatikan!
Jawaban yang benar: "Variabel perunitnya dan juga variabel secara total"
Bukan "tetap per unit" karena soal mungkin menjebak. BBL = variabel, artinya total biaya naik seiring produksi naik.

Fungsi Pengendalian Biaya FUNGSI

Jawaban: CONTROLLING (Pengendalian)

  • Planning = merencakan biaya yang akan terjadi di masa depan
  • Actuating = menggerakkan/melaksanakan rencana

High-Low Method (Metode Titik Tertinggi-Terendah) METODE
RUMUS
VC per unit (b) = (Biaya Tertinggi − Biaya Terendah) / (Unit Tertinggi − Unit Terendah)
Lalu: FC (a) = Total Cost − (VC × Unit) | Persamaan: TC = a + bQ
CONTOH SOAL — PT. SCODELARIO (SERING KELUAR!)
Bulan Januari: 1.600 unit, Total Cost Rp9.650. Bulan Februari: 1.900 unit, Total Cost Rp11.225. Tentukan persamaan biaya!
① VC/unit = (11.225 − 9.650) / (1.900 − 1.600) = 1.575 / 300 = Rp5,25/unit
② VC Total (di unit terendah) = 5,25 × 1.600 = Rp8.400
③ FC = TC − VC = 9.650 − 8.400 = Rp1.250
✅ TC = 1.250 + 5,25Q
CONTOH SOAL — SALMAN INC.
Maret: 200 unit, TC = Rp2.000. April: 600 unit, TC = Rp5.020.
① VC/unit = (5.020 − 2.000) / (600 − 200) = 3.020 / 400 = Rp7,55/unit
② VC total (unit rendah) = 7,55 × 200 = Rp1.510
③ FC = 2.000 − 1.510 = Rp490
✅ TC = 490 + 7,55Q
Metode Regresi / Least Squares REGRESI
RUMUS
TC = a + bX
dimana: a = Constant (fixed cost/intercept), b = X Coefficient (variable cost/slope), X = volume produksi
⚡ Contoh Soal Colbie-Cailat
Constant = $47.500; X Coefficient = $450; R Squared = 0,90

Persamaan biaya = TC = 47.500 + 450Q

R Squared (0,90) = goodness of fit, BUKAN dimasukkan ke persamaan!
Std. Err = ukuran kesalahan standar, BUKAN dimasukkan ke persamaan!

Process Costing — Average (Weighted Average)

Rerata Tertimbang · Unit Ekuivalen · Biaya Per Unit Ekuivalen · Cost Report
Kapan Dipakai? DASAR

Process costing dipakai saat produk diproduksi secara massal dan berkelanjutan (homogen), seperti pabrik mie, kilang minyak, pabrik semen. Tidak bisa melacak biaya per unit — kita hitung rata-rata per unit ekuivalen.

Average Method — Prinsip
  • Biaya Beginning WIP digabungkan dengan biaya periode ini
  • Semua unit dianggap "dimulai dari awal" di periode ini
  • Lebih sederhana, tidak perlu pisahkan biaya lama vs baru
  • EU = Transfer Out + (Ending WIP × %)
FIFO Method — Prinsip (Beda)
  • Biaya Beginning WIP dipisah dari biaya periode ini
  • BWIP diselesaikan dulu, baru unit baru diproses
  • Lebih akurat, tapi lebih kompleks
  • EU = Sisa BWIP + S&C + (Ending × %)
4 Langkah Process Costing Average LANGKAH
Skedul Kuantitas (Quantity Schedule)
BALANCE CHECK
Beg. WIP + Units Added = Transfer Out + Ending WIP
Jika ada nilai yang belum diketahui (???), cari terlebih dahulu dari persamaan ini!
Hitung Unit Ekuivalen (Equivalent Units)
EU — AVERAGE METHOD
EU = Transfer Out + (Ending WIP × % selesai)
Hitung TERPISAH untuk setiap elemen biaya (BBL, TKL, FOH, Transferred-In). Setiap elemen bisa punya % selesai berbeda!
Hitung Biaya per Unit Ekuivalen
COST/EU — AVERAGE (KUNCI!)
Cost/EU = (Beg. WIP Cost + Cost Added This Period) / EU
⚡ KUNCI AVERAGE: Biaya awal DIGABUNG dengan biaya periode ini di pembilang (numerator)!
Alokasikan Biaya (Cost Accounted For)
ALOKASI BIAYA
Biaya Transfer Out = Transfer Out × Total Cost/EU
Biaya Ending WIP = Σ (EU per elemen × Cost/EU per elemen)
Cek: Biaya Transfer + Biaya Ending WIP = Total Biaya Tersedia (Beg. WIP Cost + Added Cost)

⚡ Aturan Transferred-In Cost
1. TI cost diperlakukan sebagai elemen biaya TERSENDIRI (bukan BBL, TKL, atau FOH)
2. % selesai TI cost = selalu 100% (karena unit sudah selesai di dept. sebelumnya)

Process Costing — FIFO Method

First In First Out · EU FIFO · 3 Komponen Alokasi · Perbandingan dengan Average
FIFO vs Average — Perbedaan Mendasar KUNCI
Aspek AVERAGE FIFO
Perlakuan Beg. WIP Cost DIGABUNGKAN dengan biaya periode ini DIPISAHKAN dari biaya periode ini
Prioritas Penyelesaian Tidak ada — semua di-rata-rata Beg. WIP diselesaikan DULU
Rumus EU Transfer Out + (End. × %) Beg.×(1−%) + S&C + (End.×%)
Rumus Cost/EU (Beg.Cost + Added) / EU Added Cost ONLY / EU FIFO
Kerumitan Lebih sederhana Lebih kompleks, tapi lebih akurat
Alokasi biaya ke Transfer Out 1 komponen: Transfer × Cost/EU 3 komponen: BWIP completion + S&C + (tidak ada utk transfer)
⚠️ Jebakan FIFO vs Average
Dalam FIFO, Cost/EU menggunakan hanya biaya periode ini saja (tidak termasuk biaya beginning WIP)! Biaya beginning WIP dipakai terpisah untuk menghitung biaya penyelesaian BWIP.
Rumus Unit Ekuivalen FIFO — 3 Komponen RUMUS
EU FIFO = 3 KOMPONEN
EU = [Beg.WIP × (1 − % sudah selesai)] + [Transfer Out − Beg.WIP] + [End.WIP × %]
Komponen 1: Sisa pekerjaan untuk menyelesaikan BWIP | Komponen 2: Unit Started & Completed (S&C) | Komponen 3: Pekerjaan di Ending WIP
KOMPONEN 1
Menyelesaikan BWIP

Beg.WIP × (1 − % sudah selesai dari periode lalu)

Mis: BWIP=200 (40% selesai) → 200×60% = 120 EU

KOMPONEN 2
Started & Completed (S&C)

Unit yang MULAI dan SELESAI dalam periode ini = Transfer Out − Beg.WIP

Mis: Transfer=800, BWIP=200 → S&C=600 EU

KOMPONEN 3
Ending WIP

Pekerjaan yang sudah dilakukan pada unit yang belum selesai

Mis: EWIP=300 (50% selesai) → 300×50% = 150 EU

3 Komponen Alokasi Biaya Transfer Out (FIFO) ALOKASI

Dalam FIFO, biaya Transfer Out terdiri dari 3 komponen terpisah:

Biaya Penyelesaian BWIP (Completing Beginning WIP)
= Biaya BWIP (dari periode lalu) + Biaya tambahan untuk menyelesaikan sisa pekerjaan
Biaya tambahan = [Beg.WIP × (1−%) × Cost/EU periode ini] untuk setiap elemen biaya
Biaya Started & Completed (S&C)
= (Transfer Out − Beg.WIP) × Total Cost/EU periode ini
Unit yang dimulai dan diselesaikan sepenuhnya di periode ini
Total Transfer Out = Komponen 1 + Komponen 2
Kemudian: Biaya Ending WIP = EU ending × Cost/EU (untuk tiap elemen)
CEK BALANCE FIFO
Biaya BWIP + Added = (Completing BWIP + S&C) + Ending WIP
Atau: Total Transfer Out + Ending WIP = Total Biaya Tersedia
Contoh FIFO — Single Department (2 Elemen Biaya)

Dalam contoh ini, hasilnya sama karena biaya per unit BWIP = biaya per unit periode ini. Perbedaan muncul ketika biaya period lalu ≠ biaya periode ini (misalnya harga bahan baku naik). FIFO lebih akurat karena memisahkan biaya lama dan baru.

Spoiled Goods dalam Process Costing
Spoiled Goods dalam FIFO — Normal vs Abnormal TM 7 TERKAIT

Saat ada produk cacat (spoilage) di process costing, skedul kuantitas menjadi:

SKEDUL KUANTITAS DENGAN SPOILAGE
Beg.WIP + Added = Transfer Out + Ending WIP + Spoiled Units
Normal Spoilage
  • Tak terhindarkan dalam kondisi efisien
  • Biaya diserap oleh produk BAIK yang di-transfer
  • Meningkatkan cost per unit produk baik
  • Net Cost = Gross Spoilage − Scrap Value
Abnormal Spoilage
  • Bisa dicegah (kesalahan, kecelakaan)
  • Biaya diakui sebagai rugi periode (Loss)
  • Muncul di Laporan L/R
  • Tidak mempengaruhi cost per unit produk baik
NET SPOILAGE COST
Net Spoilage Cost = Gross Spoilage Cost − Scrap Value
Normal spoilage: Net cost diserap unit good output. Abnormal spoilage: Net cost = kerugian periode di L/R.

Biaya akibat cacat yang ditemukan sebelum sampai ke pelanggan

  • Re-inspeksi produk
EXTERNAL FAILURE COSTS

Biaya akibat cacat yang ditemukan sesudah sampai ke pelanggan

  • Garansi & retur produk
  • Komplain pelanggan
  • Product liability
  • Kehilangan reputasi
Jenis-Jenis Kerugian Produksi KERUGIAN
Istilah Definisi Perlakuan Akuntansi
Spoiled Goods (Produk Cacat) Unit yang tidak memenuhi standar kualitas dan tidak bisa diperbaiki secara ekonomis Normal: bebankan ke produk baik. Abnormal: rugi periode
Rework (Pengerjaan Ulang) Unit cacat yang masih bisa diperbaiki dan dijual sebagai produk normal Normal: bebankan ke FOH. Abnormal: rugi periode
Scrap (Sisa Material) Sisa bahan baku dari proses produksi yang memiliki nilai jual rendah Nilai jual scrap mengurangi biaya produksi atau FOH
Waste Material yang terbuang tanpa nilai (mis: penguapan, kebocoran) Diakui sebagai bagian dari biaya produksi normal
Normal vs Abnormal Spoilage PENTING
Normal Spoilage
  • Terjadi dalam kondisi produksi normal / efisien
  • Tidak bisa dihindari sepenuhnya
  • Biaya dibebankan ke produk baik yang lolos QC
  • Meningkatkan cost per unit produk baik
Abnormal Spoilage
  • Terjadi karena kondisi tidak normal (kesalahan, kecelakaan)
  • Seharusnya bisa dicegah
  • Biaya dibebankan sebagai rugi/kerugian periode
  • Muncul di laporan L/R sebagai kerugian

  • Transfer Cost = Transfer × Cost/EU (langsung)
  • EWIP Cost = Σ (EU_elemen × Cost/EU_elemen)
  • Tidak ada komponen S&C atau BWIP Completion
  • Lebih mudah untuk soal UTS cepat
FIFO — CHECKLIST
  • EU = BWIP×(1−%) + S&C + EWIP×%
  • Cost/EU = Added Cost ONLY / EU FIFO
  • Transfer = BWIP Completion + S&C × Cost/EU
  • EWIP Cost = Σ (EU_elemen × Cost/EU_FIFO_elemen)
  • Biaya BWIP digunakan hanya di Komponen 1
  • Lebih akurat, tapi lebih banyak langkah
⚡ Cara Cepat Mengidentifikasi Metode di Soal
Soal menyebut: "average cost flow assumption" / "rerata tertimbang" → Average Method
Soal menyebut: "FIFO" / "first in first out" → FIFO Method
Jika soal tidak menyebut dan ada Beg.WIP dengan % selesai → kemungkinan FIFO karena harus dibedakan dari unit baru.

Jebakan-Jebakan Paling Umum JEBAKAN
🪤 Jebakan Gas LPG
+

Gas LPG untuk pisang goreng = OVERHEAD PABRIK (FOH), BUKAN biaya langsung.

Yang langsung: tepung, pisang (bahan yang bisa ditelusuri per unit produk). Gas sulit diukur per unit produk, jadi masuk FOH.

🪤 Jebakan "Biaya Produksi" vs "Total Biaya Manufaktur"
+

Total Biaya Manufaktur = SEMUA pesanan yang diproses periode ini (termasuk yang masih WIP)

Kos Produksi (COGM) = Hanya yang selesai diproduksi = Transfer ke barang jadi

HPP (COGS) = Yang sudah dijual ke pelanggan

🪤 Jebakan Least Squares — R Squared
+

Dalam regresi: R Squared dan Std. Error TIDAK dimasukkan ke persamaan biaya!

Persamaan = TC = Constant + X Coefficient × Q

R² hanya menunjukkan seberapa baik model fit ke data (0.9 = 90% explained).

🪤 Jebakan Tujuan Akuntansi Biaya
+

Tujuan akuntansi biaya = informasi untuk MANAJEMEN (internal) dalam perencanaan dan pengendalian.

Bukan untuk investor, bukan untuk bank, bukan untuk petugas pajak — itu tujuan akuntansi keuangan!

🪤 Jebakan Perilaku Biaya Variabel
+

Biaya variabel: TETAP per unit, BERUBAH secara total

Bukan "menurun per unit seiring peningkatan produksi" — itu biaya tetap yang dibagi lebih banyak unit!

BBL = variabel per unit dan variabel secara total (naik bersama produksi)

🪤 Jebakan Jurnal — Transfer Barang Jadi
+

Saat produk selesai diproduksi (baru jadi, belum dijual):

Dr. Persediaan Barang Jadi | Cr. Persediaan BDP

Saat produk dijual ke pelanggan (2 jurnal):

1) Dr. Kas/Piutang | Cr. Penjualan (revenue side)

2) Dr. COGS/HPP | Cr. Persediaan Barang Jadi (cost side)

Cek saldo kuantitas terlebih dahulu

Latihan Soal UTS

Pilih jawaban untuk melihat penjelasan — mirip format UTS asli
Coba Jawab — Latihan Mandiri
Benar: 0 / 0
1. Klasifikasi biaya berdasarkan seberapa mudah suatu biaya ditelusuri ke obyeknya adalah...
💡 PENJELASAN
Berdasarkan kemudahan ditelusuri ke objek = Biaya Langsung vs Tidak Langsung. Biaya tetap/variabel = perilaku biaya. Produk/periodik = fungsi. Relevan/irrelevan = pengambilan keputusan.
2. PT. Shinobu memproses 4 pesanan: 101=Rp4.500; 102=Rp3.500; 103=Rp3.000; 104=Rp1.500. Selesai: 101,102,103. Terjual: 101,102. Berapakah BIAYA PRODUKSI (TOTAL BIAYA MANUFAKTUR)?
💡 PENJELASAN
Total Biaya Manufaktur = SEMUA pesanan yang diproses = 101+102+103+104 = 4.500+3.500+3.000+1.500 = Rp12.500. Tidak peduli sudah terjual atau belum, semua masuk total biaya manufaktur!
3. PT. Nadeko: Pesanan 201=2.500; 202=3.000; 203=2.000; 204=1.000; 205=500. Selesai: 201,202,203. Terjual: hanya 201. WIP: 204,205. Berapakah PERSEDIAAN AKHIR BARANG JADI?
💡 PENJELASAN
Ending FG = Selesai TAPI belum terjual = 202 dan 203 = 3.000+2.000 = Rp5.000. Pesanan 201 sudah jadi COGS. Pesanan 204&205 masih di WIP.
4. Informasi biaya: Beg.BB=1.300; Pembelian BB=12.500; End.BB=1.600; TKL=15.200; FOH=12.400; Beg.WIP=3.400; End.WIP=3.800; Beg.FG=2.400; End.FG=2.600. Harga Pokok Penjualan?
💡 PENJELASAN
① DM Used = 1.300+12.500−1.600 = 12.200
② Mfg Cost = 12.200+15.200+12.400 = 39.800
③ COGM = 3.400+39.800−3.800 = 39.400
④ COGS = 2.400+39.400−2.600 = Rp39.200
5. PT. Alexandra Daddario (Average): Beg.WIP=250; Added=1.250; Transfer=1.000; End.WIP=??? (50%). Biaya awal=Rp1.000.000; Biaya tambahan=Rp6.500.000. Unit Ekuivalen dan Biaya per UE adalah...
💡 PENJELASAN
Ending WIP = 250+1.250−1.000 = 500 unit
EU = 1.000 + (500×50%) = 1.000+250 = 1.250 EU
Total Cost = 1.000.000+6.500.000 = 7.500.000
Cost/EU = 7.500.000/1.250 = Rp6.000/unit
6. Informasi output regresi: Constant=$47.500; Std. Err=350; R Squared=0,90; X Coefficient=$450. Persamaan biaya adalah...
💡 PENJELASAN
Rumus regresi: TC = a + bQ dimana a=Constant, b=X Coefficient.
TC = 47.500 + 450Q
R Squared (0,90) dan Std.Err (350) TIDAK dimasukkan ke persamaan biaya!
7. PT. Scodelario: Januari: 1.600 unit, TC=Rp9.650. Februari: 1.900 unit, TC=Rp11.225. Persamaan biaya (High-Low Method)...
💡 PENJELASAN
b = (11.225−9.650)/(1.900−1.600) = 1.575/300 = 5,25
VC total = 5,25 × 1.600 = 8.400
FC = 9.650 − 8.400 = 1.250
TC = 1.250 + 5,25Q
8. Entri jurnal untuk mencatat transfer produk yang telah selesai diproduksi (siap dijual ke konsumen) adalah...
💡 PENJELASAN
Produk selesai diproduksi = pindah dari WIP ke Finished Goods:
Dr. Persediaan Barang Jadi | Cr. Persediaan BDP
Belum ada penjualan di sini, jadi Penjualan/Kas belum diakui.
9. Alicia Vikander (Average): Beg.WIP=200; Added=1.800; Transfer=1.500; End.WIP=500 (BBL 80%, Konversi 60%). TOTAL UNIT EKUIVALEN BBL dan Konversi...
💡 PENJELASAN
EU BBL (Average) = Transfer Out + (End.WIP × 80%) = 1.500 + (500×80%) = 1.500+400 = 1.900
EU Konversi = 1.500 + (500×60%) = 1.500+300 = 1.800
10. PT. Suruga: P101=2.500; P102=3.000; P103=1.000. Selesai: P101,P102. Terjual: P101. Revenue=7.500; Biaya penjualan=1.000; Biaya admin=750. Laba Operasi?
💡 PENJELASAN
Revenue = 7.500
COGS = Pesanan 101 = 2.500
Gross Profit = 7.500−2.500 = 5.000
Op. Expenses = 1.000+750 = 1.750
Laba Operasi = 5.000−1.750 = Rp3.250
